## Prerequisites

- Oracle Cloud Always Free ARM instance (Ampere A1) **or** Render.com account
- Ubuntu 22.04/24.04 aarch64 (Oracle) or Docker (Render)
- Root access (Oracle) or admin access (Render)
- Nokia G-2425G-A router configured as access point
- Android phone with SMS monitor app

## Option A: Oracle Cloud (Primary — Recommended)

### Step 1: Create Oracle Cloud Account & Instance

1. **Sign up** at [cloud.oracle.com](https://cloud.oracle.com)
   - Requires a credit card for verification (no charges on Always Free tier)
   - Select **Af-Johannesburg-1** region (closest to Kenya)

2. **Create an Always Free ARM instance:**
   - Go to **Compute → Instances → Create Instance**
   - Name: `wifi-hotspot`
   - Image: **Ubuntu 22.04 Minimal aarch64** (or 24.04)
   - Shape: **VM.Standard.A1.Flex** (Ampere A1)
   - OCPUs: **2** (max free), RAM: **12 GB**
   - Boot volume: **200 GB** (max free)
   - **Networking:** Create/select a VCN and public subnet
   - **SSH keys:** Generate or upload your public key
   - Click **Create**

3. **Open required ports** in Security List:
   - Go to **Networking → Virtual Cloud Networks → your VCN → Security Lists → Default**
   - Add Ingress Rules:
     | Port | Protocol | Source | Purpose |
     |------|----------|--------|---------|
     | 22 | TCP | 0.0.0.0/0 | SSH |
     | 80 | TCP | 0.0.0.0/0 | HTTP (captive portal) |
     | 443 | TCP | 0.0.0.0/0 | HTTPS |
     | 53 | TCP+UDP | 0.0.0.0/0 | DNS (captive portal) |

4. **Note your public IP** from the instance details page.

### Step 2: Connect & Upload

```bash
# Connect via SSH
ssh -i ~/.ssh/oracle_key ubuntu@<your-oracle-ip>

# On your local machine, upload deployment files
scp -i ~/.ssh/oracle_key -r deployment/ ubuntu@<your-oracle-ip>:/tmp/wifi-hotspot-deploy/

# On the server, move to a proper location
sudo mv /tmp/wifi-hotspot-deploy /root/wifi-hotspot-deploy
sudo chown -R root:root /root/wifi-hotspot-deploy
```

### Step 3: Configure Environment

```bash
cd /root/wifi-hotspot-deploy

# Copy template and edit
cp .env.template .env
nano .env
```

**Required values to set:**

| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://hotspot_user:CHANGEME@localhost/wifi_hotspot` |
| `SECRET_KEY` | Flask secret key (random 64 hex chars) | Auto-generated by deploy.sh |
| `API_KEY` | API authentication key | Auto-generated by deploy.sh |
| `ROUTER_IP` | Your Nokia router's LAN IP | `192.168.1.1` |
| `ROUTER_USER` | Router admin username | `admin` |
| `ROUTER_PASS` | Router admin password | Your router password |
| `OWNER_PHONE` | Your phone number for SMS alerts | `0712345678` |
| `DOMAIN` | Your public IP or domain | `129.153.xx.xx` |

### Step 4: Run Deployment

```bash
cd /root/wifi-hotspot-deploy
chmod +x deploy.sh
sudo bash deploy.sh
```

**The script will automatically:**
1. Install system packages (Python, PostgreSQL, Nginx, dnsmasq, fail2ban)
2. Configure firewall (iptables)
3. Set up fail2ban for SSH brute-force protection
4. Create PostgreSQL database with schema + seed data
5. Deploy Flask app with Gunicorn
6. Configure Nginx (captive portal detection + reverse proxy)
7. Set up dnsmasq for captive portal DNS hijacking
8. Configure SSL (Let's Encrypt if domain, self-signed if IP-only)
9. Create systemd services (API + Scheduler)
10. Install logrotate config
11. Output generated credentials

### Step 5: Save Credentials

After deployment, credentials are saved to `/root/.wifi-hotspot-credentials`.

```bash
cat /root/.wifi-hotspot-credentials
```

**⚠️ Save these immediately — they won't be shown again!**

Contents:
```
DB_USER=hotspot_user
DB_PASS=<generated>
SECRET_KEY=<generated>
API_KEY=<generated>
SERVER_IP=<your-ip>
```

### Step 6: First-Run VIP Setup

The admin dashboard is accessible at `http://<your-ip>/admin`. Default credentials:

- **Username:** `admin`
- **Password:** `admin123`

**⚠️ Change the admin password immediately after first login!**

To manually set a VIP user (permanent access):
```bash
sudo -u postgres psql -d wifi_hotspot -c \
  "INSERT INTO users (mac_address, phone_number, plan_type, is_permanent)
   VALUES ('AA:BB:CC:DD:EE:FF', '0712345678', 'vip', TRUE)
   ON CONFLICT (mac_address) DO UPDATE SET plan_type='vip', is_permanent=TRUE;"
```

### Step 7: Verify Deployment

```bash
# Full health check
bash /root/wifi-hotspot-deploy/monitor.sh

# Quick API check
curl http://localhost:8000/health

# Check all services
systemctl status wifi-hotspot-api
systemctl status wifi-hotspot-scheduler
systemctl status dnsmasq
systemctl status nginx

# Check active sessions
sudo -u postgres psql -d wifi_hotspot -c "SELECT * FROM v_active_sessions;"

# Check revenue
sudo -u postgres psql -d wifi_hotspot -c "SELECT * FROM v_revenue_daily LIMIT 30;"
```

### Step 8: Configure Router

On your **Nokia G-2425G-A** router:

1. Log into router admin (usually `http://192.168.1.1`)
2. Go to **Network → LAN Settings**
3. Set **DNS Server** to your Oracle Cloud IP (e.g., `129.153.xx.xx`)
4. Go to **Wireless → Basic Settings**
5. Set SSID name (e.g., `WiFi-Hotspot`)
6. **Disable** the router's built-in DHCP if dnsmasq handles DHCP
   - Or configure DHCP to advertise your server as DNS: `dhcp-option=option:dns-server,<your-ip>`

## Option B: Render + Neon.tech (Fallback)

Use this if Oracle Cloud is unavailable or you prefer a managed platform.

### Step 1: Create Neon.tech Database

1. **Sign up** at [neon.tech](https://neon.tech)
2. **Create a project:**
   - Name: `wifi-hotspot`
   - Region: Choose closest to your users (e.g., `US East` or `EU Frankfurt`)
3. **Create a database:**
   - Database name: `wifi_hotspot`
4. **Get connection string:**
   - Go to **Dashboard → Connection Details**
   - Copy the connection string (format: `postgresql://user:pass@host/db?sslmode=require`)
   - Save this — you'll need it for Render

5. **Initialize schema:**
   ```bash
   # Install psql if needed
   sudo apt-get install -y postgresql-client

   # Connect and run schema
   psql "postgresql://user:pass@host/wifi_hotspot?sslmode=require" -f setup_db.sql
   ```

### Step 2: Deploy to Render

1. **Sign up** at [render.com](https://render.com)
2. **Create a Web Service:**
   - Connect your GitHub repo (or use "Deploy from existing image")
   - Name: `wifi-hotspot-api`
   - Runtime: **Python 3**
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn -c gunicorn.conf.py wsgi:app`
   - Instance Type: **Starter** ($7/mo) or **Free** (sleeps after 15min)

3. **Set Environment Variables** in Render dashboard:

   | Key | Value |
   |-----|-------|
   | `DATABASE_URL` | Your Neon.tech connection string |
   | `SECRET_KEY` | Generate: `openssl rand -hex 32` |
   | `API_KEY` | Generate: `openssl rand -hex 24` |
   | `FLASK_ENV` | `production` |
   | `LOG_LEVEL` | `INFO` |
   | `ADMIN_USER` | `admin` |
   | `ADMIN_PASS` | `admin123` (change after first login) |
   | `ROUTER_IP` | Your Nokia router's public/VPN IP |
   | `ROUTER_USER` | `admin` |
   | `ROUTER_PASS` | Your router password |
   | `OWNER_PHONE` | Your phone number |

4. **Create a Background Worker** (for the scheduler):
   - Name: `wifi-hotspot-scheduler`
   - Same repo
   - Start Command: `python scheduler.py`
   - Same environment variables

### Step 3: Custom Domain (Optional)

1. In Render, go to your web service → **Settings → Custom Domains**
2. Add your domain (e.g., `hotspot.yourdomain.com`)
3. Update DNS A record to point to Render's IP
4. SSL is automatic on Render

### Step 4: First-Run VIP Setup

Same as Oracle Cloud — access admin dashboard at your Render URL and set up VIP users.

### Limitations of Render + Neon.tech

- **No dnsmasq:** Captive portal DNS hijacking won't work on Render (shared infrastructure)
- **No direct router control:** Router sync requires VPN or public IP to router
- **Cold starts on free tier:** App sleeps after 15min of inactivity
- **No SMS monitoring:** Android phone SMS bridge needs a stable server connection
- **Best for:** Admin dashboard, API backend, payment processing — NOT for captive portal itself

## Captive Portal Flow

```
1. Client connects to WiFi (Nokia router)
2. Client's DNS queries → dnsmasq → resolves ALL domains to server IP
3. Client tries to access internet → hits Nginx on port 80
4. Nginx returns portal page (or redirects to portal)
5. Client enters voucher code or pays via M-Pesa
6. API activates session → scheduler syncs MAC to router
7. Router allows client traffic → internet access granted
8. Session expires → scheduler removes MAC from router
```

## SSL / Let's Encrypt

If you have a domain:
1. Point DNS A record to your Oracle Cloud IP
2. The deploy script will auto-run certbot
3. Auto-renewal is configured via cron

If using IP only:
- Self-signed certificate is generated
- Browsers will show warning (acceptable for captive portal)
- Consider using a free domain (e.g., DuckDNS) for proper SSL

## Security

### Firewall Rules (iptables)
- Port 22: SSH (rate-limited: 3 attempts/60s)
- Port 80: HTTP (captive portal detection)
- Port 443: HTTPS
- Port 53: DNS (captive portal)
- All other ports: DROP

### Fail2ban
- SSH: 3 failed attempts → 1 hour ban
- Logs: /var/log/auth.log

### Application Security
- API key authentication on all API endpoints
- Rate limiting via Nginx (30 req/s API, 5 req/min auth)
- PostgreSQL only accessible from localhost
- Systemd service hardening (NoNewPrivileges, ProtectSystem, PrivateTmp)

## Router Configuration (Nokia G-2425G-A)

The router should be configured as:
1. **Access Point mode** (not router mode if server handles DHCP)
2. **DNS server** pointed to your Oracle Cloud IP
3. **DHCP** either disabled (dnsmasq handles it) or configured with DNS pointing to server

## Troubleshooting

### Captive portal not showing on iOS
```bash
# Check nginx captive portal detection
curl -v http://<server-ip>/hotspot-detect.html
# Should return 302 redirect (NOT 200 "Success")

# Check dnsmasq is running and on port 53
systemctl status dnsmasq
ss -ulnp | grep :53

# Test DNS resolution from client device
nslookup example.com <server-ip>
```

### Captive portal not showing on Android
```bash
# Android uses /generate_204 — should return 302
curl -v http://<server-ip>/generate_204
```

### Sessions not expiring
```bash
# Check scheduler
systemctl status wifi-hotspot-scheduler
journalctl -u wifi-hotspot-scheduler -f

# Manual expire check
sudo -u postgres psql -d wifi_hotspot -c \
  "UPDATE sessions SET is_active=FALSE WHERE expires_at < NOW() AND is_active=TRUE;"
```

### Router sync not working
```bash
# Check router connectivity
ping 192.168.1.1

# Check sync log
sudo -u postgres psql -d wifi_hotspot -c \
  "SELECT * FROM payments WHERE status='confirmed' ORDER BY created_at DESC LIMIT 10;"
```

### DNS resolution broken on server
```bash
# Ensure /etc/resolv.conf uses 8.8.8.8 (NOT 127.0.0.1)
cat /etc/resolv.conf
# Should show: nameserver 8.8.8.8

# If not, fix it:
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
```
